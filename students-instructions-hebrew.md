# project-express-server-week-4

## מטרות הפרויקט

בפרויקט הזה תתרגלו את הכישורים הבאים:

- שיטות בסיסיות בשרת HTTP
- עבודה עם החלקים המרכזיים של HTTP, כולל requests, responses, headers ו-cookies
- הבנת HTTP routing
- שימוש ב-query params כדי לסנן נתונים
- JSON parsing
- יסודות Express
- Express basic auth
- Express middleware וטיפול בשגיאות

## תיאור הפרויקט

הפרויקט הזה ידמה תהליך backend פשוט עבור חנות אונליין.

אתם יכולים לבחור איזה סוג של חנות אונליין תרצו ליצור. לדוגמה:

- חנות בגדים
- חנות נעליים
- חנות צעצועים
- חנות ספרים

## דרישות הפרויקט

### המשתמש צריך להיות מסוגל:

- להירשם / להתחבר.
- לשלוח בקשות לשרת בלי להתחבר מחדש, כל עוד יש לו auth token שלא פג תוקפו.
- לראות את רשימת המוצרים הזמינים בחנות. הבקשה הזאת לא דורשת authentication.
- להוסיף מוצר חדש לעגלה שלו. בנוסף, הוא יכול גם להסיר פריטים מהעגלה.
- לראות את רשימת כל המוצרים שנמצאים בעגלה שלו.
- לבצע checkout ולשלם על ההזמנה בעזרת הכסף שיש לו ביתרה.
- לראות את סכום הכסף הנוכחי שיש לו ביתרה.

### כללים בסיסיים של הפרויקט:

- כל משתמש צריך לקבל סכום כסף קבוע בחשבון שלו בזמן ההרשמה.
- המשתמש יכול לקנות רק מוצרים שקיימים במלאי. במילים אחרות, אם מוצר לא נמצא במלאי, אי אפשר להוסיף אותו לעגלה.
- עם זאת, המשתמש יכול לראות את כל המוצרים בחנות גם אם הם לא זמינים במלאי, כל עוד ברור שהם לא זמינים.
- המשתמש יכול לשלם על הזמנה רק עם סכום הכסף שיש לו ביתרה. ניסיון רכישה של משתמש שאין לו מספיק כסף צריך להיכשל ולהחזיר הודעת שגיאה למשתמש.
- המשתמש לא יכול להוסיף או לשנות את היתרה שלו.
- אם המשתמש מנסה לשלוח בקשה עם auth token שפג תוקפו, הבקשה צריכה להידחות עם הודעת שגיאה מתאימה, ולא עם שגיאת "unauthorized" כללית.
- המוצרים לא נוצרים על ידי משתמש כלשהו. במקום זאת, הם צריכים להיות מוגדרים מראש על ידיכם.
- מידע רגיש, כמו פרטי התחברות ל-DB, ובמקרה שלנו הנתיב לקובץ ה-JSON שמדמה את ה-DB, לא צריך להיות כתוב ישירות בקוד. במקום זאת, הוא צריך להישמר בקובץ environment variables שלא נכנס ל-source control, למשל `.env`.

### משימת למידה עצמית:

אתם חייבים להצפין את הסיסמה של המשתמש בזמן ההרשמה באמצעות bcryptjs. בזמן ההתחברות, אתם חייבים להשוות בין הסיסמה שנשלחת לשרת לבין הסיסמה ששמורה ב-database, כלומר בקובץ ה-JSON שמדמה את ה-DB.

### דרישות טכניות של הפרויקט:

הפרויקט חייב להשתמש בטכנולוגיות / ספריות הבאות:

- Express.js
- JavaScript נקי, ללא TypeScript
- Node.js
- bcryptjs, כחלק ממשימת הלמידה העצמית
- Git

### הערות חשובות:

- הדרך היחידה לתקשר עם השרת היא דרך הטרמינל או דרך API client כמו Postman.
- מותר לכם לבנות כמה routes שתרצו. עם זאת, הציון של הפרויקט יושפע בצורה משמעותית מאיכות ה-routes שלכם ומהשאלה האם הם בנויים נכון.
- כדאי ליצור מבנה תיקיות ומבנה קוד נקיים וברורים לפרויקט.
- באותה צורה, ככל שתנהלו את קוד הפרויקט טוב יותר עם Git, כך הפרויקט יהיה טוב יותר.
- עם כל זה, הדבר הראשון שאתם חייבים לוודא הוא שהקוד עובד. זה הגורם המרכזי.
